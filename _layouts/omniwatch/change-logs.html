Track all changes within your OmniWatch installation

OmniWatch stores a record of all modifications within its change logs

Each installation of OmniWatch is designed and set up before going live with the input of both Spook and the client to ensure that the correct monitoring is in place from day one.

However, it's not unusual for the starting configuration of OmniWatch to change and evolve over time and it is important to be able to track what alterations have been made, by who and the reason for it.

Hardware changes

One of the key parts of OmniWatch is that everything is customisable. This includes the ability to set individual sensor trip values, state if a device/alarm should be escalated or not, or make the sensor informaitonal only.

Each change can have a huge impact on the operation and flow of OmniWatch and when there are many users administering an installation or split across day/night shifts or even timezones important changes can be missed by others. Within OmniWatch every trip value, critical and active status change are logged and stored allowing for a quick communication of modifications. These changes can also be emailed out to management and key staff memebers.

Rota changes

Some organisations have a change of on-call staff associated with a rota change on a specific day or date. This is achieved by simply making the current rota inactive and the new rota active with a single button within the contact function. Management can check or even manage the changes themselves. As with all changes to the Spook set-up, these are captured within the change logs.

Email notifications of changes

With large installations or companies that are split over multiple timezones it can often be difficult to track and monitor changes, and the reasons behind them, made by other members of staff. With OmniWatch it is possible to set up contacts to receive email notifications of any changes within realtime, keeping you up to date without having to log in or search through logs online.

Searchable and sortable

Over time many changes can be actioned on OmniWatch which can cause the amount of recorded items to be substantial. Users are able to search within the change logs using any data to filter down to the changes they wish to see.

For example, simply type temp for all changes associated with temperature; or type in the name of a contact to track all amendments associated with that staff member.

Included on each item page (contact/sensor)

Each view of a customisable device (be it an invidual sensor, piece of monitoring hardware or an entire installation) OmniWatch presents all relevent changes on the same page.

Exported with reports

When creating reports OmniWatch includes a table of all changes performed on that installation within the selected time frame, giving the perfect snapshot of what happened.
