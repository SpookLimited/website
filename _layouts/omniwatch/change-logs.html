Event log

Each installation of OmniWatch is designed and set up before going live with the input of both Spook and the client to ensure that the correct monitoring is in place from day one. However, it's not unusual for the starting configuration of OmniWatch to change and evolve over time and it is important to be able to track what alterations have been made, by who and the reason for it.

As there are many customisable parts to OmniWatch a user can change matters in-line with how they want OmniWatch to work, therefore it is important to log these for future reference.

Rota changes

Some organisations have a change of on-call staff associated with a rota change on a specific day or date. This is achieved by simply making the current rota inactive and the new rota active with a single button within the contact function. Management can check or even manage the changes themselves. As with all changes to the Spook set-up, these are captured within the event log.

Email notifications of changes

Contacts are able to be set up to receive instant notifications of any changes to their monitoring installation. This means users are always aware of what is happening within their set up without the need to constantly check online.

Searchable and sortable

Over time many changes can be actioned on OmniWatch which can cause the amount of recorded items to be substantial. Users are able to search within the event log using any data to filter down to the events they wish to see.

For example, simply type temp for all changes associated with temperature; or type in the name of a contact to track all amendments associated with that staff member.
