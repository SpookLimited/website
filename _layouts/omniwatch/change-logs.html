Track all changes within your OmniWatch installation

OmniWatch stores a record of all modifications within its event logs

OmniWatch installations evolve over time. It's not unusual for the starting configuration of OmniWatch to change so it is important to be able to track what alterations have been made, by whom and for what reason.

Hardware changes

OmniWatch is customised to a users requirements. This includes the ability to set individual sensor trip values, state if a device/alarm should be escalated or not, or make the sensor informational only.

Each change can have a huge impact on the operation and flow of OmniWatch and when there are many users administering an installation or split across day/night shifts or even timezones important changes can be missed by others.

OmniWatch ensures every sensors details are logged and stored accurately.

Contact management audits

Many organisations have an on-call rota system for teams or individuals to assume the first line of support for alerts. Changes to rotas and emergency staff details are recorded in the event log.

Email notifications of changes

With large installations or companies that are split over multiple timezones it can often be difficult to track and monitor changes, and the reasons behind them. OmniWatch can send email notifications of any changes in realtime, keeping staff up to date without having to log in or search through logs online.

Global search

Over time many changes can be applied to OmniWatch which can cause the amount of recorded events to be substantial. Users are able to search for changes easily in the free text search engine.

For example, simply type temp for all changes associated with temperature; or type in the name of a contact to track all amendments associated with that staff member.
